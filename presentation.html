Proposal

proposal.md

Establishes why. What problem it solves, why now, what changes, who owns it. One page. Forces you to articulate the idea before building it.

Specs

specs/<workstream>/spec.md

Defines what done looks like. GIVEN/WHEN/THEN scenarios. Every criterion is verifiable with a clear pass/fail. No ambiguity.

Design

design.md

Explains how. Context, goals, approach, tools, risks, coordination. Optional -- create only when complexity warrants it.

Tasks

tasks.md

Breaks work into assignable units. Checkbox format for tracking. Each task completable in under 30 minutes. Grouped by phase with verification.
